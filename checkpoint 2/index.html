App architecture

The app uses an MVC-like architecture. The controller is a Node.js program hosted on an Amazon EC2 instance. Another controller, an Angular.js program, is delivered to the client; this controller does the brunt of the work, dealing with Firebase no-SQL data store and Amazon S3 image storage. The data (model) is stored and pulled from Firebase on the client. All the views delivered to the client, and are accessed by the Angular program.

Basic room and image texture mapping

Graphics are done in three.js. Note that the room is colored green to indicate that materials and textures are still in prototype phase. Also notice "axes" drawn in order to easily tell locations in Euclidean space.

The plain room is a scene with several polygons: a plane for the "ground", a four identical rectangle blocks with a small depth and bevel. The plane is of large yet discrete size.

Light effects are achieved using directional, ambient, and point lights. Directional lights illuminate surfaces with the same diffuse light vector, and allow for shadows on affected surfaces. Point lights create the effect of a localized light, or "lightbulb". Surfaces, as of now, have a specular quality. This may change after the advanced materials are programmed. All lights are affected by a global dimming effect after a certain distance (three.js "fog").

Texture mapping is accomplished by using the three.js TextureLoader object, and mapping to a plane polygon (proportional to image dimensions) using a UV map and NEAREST strategy for selecting texture points.
